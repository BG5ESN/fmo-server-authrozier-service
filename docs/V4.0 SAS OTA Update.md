# V4.0 SAS OTA Update

## 1. 概述

SAS 启动时自动检查远端最新版本，根据部署环境选择更新策略：

| 部署环境 | 更新策略 |
|----------|----------|
| 裸机（Windows / Linux / macOS） | `sas --update` 自动下载 + 脚本自替换 + 重启 |
| Docker | 日志提示 `docker pull` 命令，外部工具负责更新 |

版本检查为异步非阻塞，网络异常静默跳过，不影响正常启动。

---

## 2. 版本号

| 项 | 说明 |
|----|------|
| 格式 | semver `major.minor.patch`，如 `4.2.0` |
| 定义位置 | `src/Sas.csproj` → `<Version>4.2.0</Version>` |
| 运行时读取 | `AssemblyInformationalVersionAttribute` |
| 比较方式 | `System.Version` 标准比较 |

## 3. 远端版本清单

URL：`https://bg5esn.com/share/fmo/sas.json`

```json
{
  "version": "4.2.1",
  "notes": "Bug fixes and performance improvements.",
  "url": "https://github.com/bg5esn/fmo-sas/releases/latest",
  "assets": {
    "win-x64":      "https://github.com/bg5esn/fmo-sas/releases/download/v4.2.1/sas-win-x64.zip",
    "linux-x64":    "https://github.com/bg5esn/fmo-sas/releases/download/v4.2.1/sas-linux-x64.tar.gz",
    "linux-arm64":  "https://github.com/bg5esn/fmo-sas/releases/download/v4.2.1/sas-linux-arm64.tar.gz",
    "osx-arm64":    "https://github.com/bg5esn/fmo-sas/releases/download/v4.2.1/sas-osx-arm64.tar.gz",
    "osx-x64":      "https://github.com/bg5esn/fmo-sas/releases/download/v4.2.1/sas-osx-x64.tar.gz"
  }
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `version` | string | 是 | 最新 semver 版本号 |
| `notes` | string | 否 | 更新说明，展示给用户 |
| `url` | string | 否 | 发布页 / GitHub Releases 链接 |
| `assets` | object | 否 | 各平台下载地址，key 为 .NET Runtime Identifier |

`assets` 的 key 对应 .NET 的 `RuntimeInformation.RuntimeIdentifier`，SAS 运行时按当前 RID 匹配对应下载链接。

### 3.1 本地配置

SAS 通过 `config.json` 中的 `update.enabled` 控制是否启用版本检查：

```json
{
  "update": { "enabled": true }
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `update.enabled` | bool | `true` | 是否在启动时检查远端版本。设为 `false` 可完全关闭 |

旧配置文件中若不存在 `update` 节，默认视为启用。

## 4. 启动时版本检查

```
SAS 启动
  ├─ 读取本地版本号
  ├─ 日志输出版本
  ├─ HTTP 监听器启动
  └─ config.update.enabled == true
       └─ 异步请求 sas.json（5 秒超时，不阻塞）
            ├─ 成功 → 解析 JSON
            │    ├─ 本地 < 远端
            │    │    ├─ 裸机 → [Warn] 新版本 + --update 提示
            │    │    └─ Docker → [Warn] 新版本 + docker pull 提示
            │    └─ 本地 >= 远端 → 无输出
            └─ 失败 → 静默（网络不通 / 超时 / JSON 格式错误）
```

### 4.1 环境检测

| 检测条件 | 判定环境 |
|----------|----------|
| `Environment.GetEnvironmentVariable("DOTNET_RUNNING_IN_CONTAINER") == "true"` | Docker |
| `File.Exists("/.dockerenv")` | Docker |
| 其他 | 裸机 |

### 4.2 日志输出

**裸机环境，有新版本：**
```
[Warn] New SAS version available: 4.2.1 → https://github.com/anomalyco/fmo-sas/releases/latest
[Warn]   更新说明：Bug fixes and performance improvements.
[Warn]   Update now: sas --update
```

**Docker 环境，有新版本：**
```
[Warn] New SAS version available: 4.2.1 → https://github.com/anomalyco/fmo-sas/releases/latest
[Warn]   更新说明：Bug fixes and performance improvements.
[Warn]   Update: docker pull fmo/sas:latest && docker compose up -d
```

**版本已是最新或无网络**：不输出任何内容。

> **关闭版本检查**：在 `config.json` 中设置 `"update": { "enabled": false }` 即可关闭启动时的版本检查。

## 5. `--update` 命令

### 5.1 裸机流程

```
sas --update

1. 请求 sas.json → 获取 version + assets[当前RID]
2. 比对本地版本：本地 >= 远端 → "已是最新版本" 退出
3. 创建临时目录 %TEMP%/sas_update/
4. 下载 zip/tar.gz 到临时目录
5. 解压到临时目录
6. 验证解压产物存在（Sas / Sas.exe）
7. 写入更新脚本
8. 打印 "更新就绪，即将重启…"
9. 启动更新脚本（传入当前进程 PID + 新文件路径）
10. 退出 SAS 进程

更新脚本：
  ├─ 轮询等待 SAS 进程退出
  ├─ 移动新文件覆盖旧文件
  ├─ 启动新版本 SAS
  ├─ 清理临时目录
  └─ 删除自身
```

### 5.2 Docker 流程

```
sas --update

[Info] Running in Docker. Update with:
[Info]   docker pull fmo/sas:latest && docker compose up -d
```

Docker 环境不执行自更新。容器更新由外部工具负责。

## 6. 更新脚本

### 6.1 Windows (`update.bat`)

SAS 调用：`cmd /c update.bat <PID> <新文件目录>`

```batch
@echo off
setlocal
set PID=%1
set SRCDIR=%2

:wait
timeout /t 1 /nobreak >nul
tasklist /fi "PID eq %PID%" 2>nul | findstr "%PID%" >nul
if not errorlevel 1 goto wait

move /y "%SRCDIR%\Sas.exe" "%~dp0Sas.exe"
rmdir /s /q "%SRCDIR%"
start "" "%~dp0Sas.exe"
del "%~f0"
```

### 6.2 Linux / macOS (`update.sh`)

SAS 调用：`sh update.sh <PID> <新文件目录>`

```sh
#!/bin/sh
PID=$1
SRCDIR=$2
DESTDIR=$(dirname "$0")

while kill -0 "$PID" 2>/dev/null; do
    sleep 1
done

mv "$SRCDIR/Sas" "$DESTDIR/Sas"
chmod +x "$DESTDIR/Sas"
rm -rf "$SRCDIR"
"$DESTDIR/Sas" &
rm "$0"
```

## 7. 错误处理

| 阶段 | 失败原因 | 行为 |
|------|----------|------|
| 请求 sas.json | 网络不通 / 超时（5 秒） | 静默跳过 |
| 启动时版本检查 | JSON 格式错误 | 静默跳过 |
| `--update` 比对版本 | 本地 >= 远端 | 提示 `Already up to date.` 退出 |
| `--update` 下载 | HTTP 4xx/5xx / 超时 | 提示 `Download failed: <原因>` 退出 |
| `--update` 解压 | 文件损坏 / 格式错误 | 提示 `Verification failed.` 清理临时目录退出 |
| `--update` 验证 | 解压后无 Sas/Sas.exe | 同上 |
| 更新脚本执行 | 替换失败 | 旧文件保留，不影响运行 |
| 新版本启动 | 配置不兼容 / crash | 外部 supervisor（systemd / Docker restart policy）接管 |

**不设自动回滚。** 若新版本无法启动，用户手动替换旧二进制即可。

---

## 8. 安全考量

1. **HTTPS only**：sas.json 和 assets 下载均通过 HTTPS。
2. **无签名校验（当前版本）**：assets 通过 HTTPS 的传输层完整性保护，不额外校验文件签名。GitHub Releases 页面本身提供 SHA256 checksum，用户可按需手动校验。
3. **临时目录**：更新产物写入 `%TEMP%/sas_update/`，更新完成后清理。
4. **无自动更新**：不静默下载或替换，必须通过 `--update` 明确触发。
5. **误用防护**：Docker 环境下 `--update` 无效，避免容器内意外修改只读文件系统。
